Thank you for your interest in contributing! This document provides guidelines for contributing.

## Ways to Contribute

- Report bugs or suggest features via GitHub Issues
- Improve documentation
- Submit bug fixes or new features via Pull Requests
- Help answer questions from other users

## Getting Started

1. Fork the repository
2. Follow the [Development Guide](docs/development.md) for setup
3. Create a branch for your changes
4. Make your changes with clear commit messages
5. Test your changes thoroughly
6. Submit a Pull Request

## Code Standards

- Follow the coding conventions in [Development Guide](docs/development.md)
- Ensure type safety with TypeScript and Zod validation
- Write clear, descriptive commit messages
- Update documentation for new features
- Test on the target platform (Raspberry Pi OS if possible)

## Pull Request Process

1. Update documentation as needed
2. Ensure all tests pass
3. Update the [changelog](docs/changelog.md)
4. Request review from maintainers
